# Contributing to Project A.L.I.S. 🛠️

First off, thank you for considering contributing to Project A.L.I.S.! It’s people like you who make the engineering community better by reducing waste and optimizing processes.

## 🚀 How Can I Contribute?

### Reporting Bugs
* Use the GitHub Issues tab to report bugs.
* Describe the steps to reproduce the error and include your operating system.

### Suggesting Features
* We are always looking for new Lean metrics or better ways to visualize Six Sigma data. 
* Open an issue first to discuss your idea before writing code.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure your code is formatted using `cargo fmt`.
3. If you've added a new mathematical module, please include a test case.
4. Issue your PR with a clear description of the "Waste" you are helping to eliminate.

## 📐 Coding Standards
* **Safety First:** This is a Rust project. Avoid `unsafe` blocks unless absolutely necessary.
* **Lean Logic:** Every feature should serve the goal of helping engineers find the most efficient route.
